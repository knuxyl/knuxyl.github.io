Dragonframes

February 3, 2024

https://github.com/knuxyl/dragonframes#

A while back I made an addon for World of Warcraft 3.3.5a called DragonFrames. It changes the portrait art around the player and/or enemy icon in the top left to whatever they want.

I extracted the original assets from newer World of Warcraft clients and the original, converted them from blp to png, and ran them through an AI upscaler

realcugan-ncnn-vulkan -s 4 -n 3 -i "input.png" -o "output.png"

I included the option for the player to have the art dynamically change depending on the level. The entire project was coded in Lua, which is WoW’s native addon code.

The problem I ran into was a serious lack of documentation on the API, and the API that was documented did not behave as expected. The code is a mess because of this. I did what I could to get this to work. I remember the biggest problem was finding the right callbacks/event handlers, and the events themselves. I believe this still had the bug that if the UI is refreshed, it would show white boxes instead of the textures. I couldn’t find a callback to reload the textures.

There are libraries that I could have used to simplify some of this, but because this is an old version of the game, the documentation for these version is just as lacking.

I created this addon because I always wanted this feature because of how rare it was to see these frames. It’s working and is no where near perfect.
